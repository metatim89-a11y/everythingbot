# GEMINI Instructions: everythingbot

This document serves as the foundational mandate for all AI interactions within the **everythingbot** project. It takes absolute precedence over general defaults.

## Project Vision: "$ide$Fi$ale$"
A high-performance NLP automation system utilizing a localized Phi-1.5 model, session-isolated file CRUD, and a highly customized "5idesFi5ales-OS" React UI.

## Scope Control & Approval Mandate (Strict Protocol)
To maintain stability and prevent "progress erasure," the AI must adhere to these rules:
1. **One Objective at a Time:** Focus exclusively on the current approved task. Do not multi-task or introduce unrelated refactors.
2. **Explicit Approval Only:** NEVER modify code or files that have not been specifically discussed and approved by 5idesFi5ales during the current session.
3. **No "Thinking Ahead":** Do not implement features or fixes based on assumptions of future needs. If an improvement is identified, it must be discussed first.
4. **Deferred Ideas Management:** If a suggestion is made by the AI but not approved for immediate implementation, the AI must store that idea and remind the user at a later, more appropriate time.

## Architecture: The 4 Pillars & 2 Trunks
Maintain strict domain isolation. NEVER cross-contaminate logic between pillars:

### The 4 Pillars (Active Services)
- **`[A] /web` (The Face):** React SPA and Tailwind styles (Pure frontend).
- **`[B] /server` (The Node Brain):** ExpressJS/MCP backend handling API routes and database reads.
- **`[C] /app` (The Python Brain):** Python NLP Service running local `Phi-1.5` models and heavy AI logic.
- **`[D] /worker` (The Hands):** Puppeteer/Playwright scrapers, bots, and automations.

### The 2 Trunks (Passive Storage)
- **`[E] /data` & `/models` (The Memory):** SQLite databases, file uploads, and downloaded `tar.gz` or `.gguf` AI weights.
- **`[F] .hidden` (The Vault):** All `.sstar` snapshots, automated logs, `LOGS/`, and raw system output. (Keep the root clean).

## Execution Standards
### 1. Unified Environment (Debian WSL)
To prevent path-crossing, filesystem confusion, and script failures, ALL project code, shell scripts, and terminal commands MUST be executed strictly within the **Debian WSL terminal environment**. Do not use standard Windows PowerShell or Command Prompt for execution.

## Development Standards
### 1. File Delivery and Editing Protocol
Because the AI is now operating directly within the development environment rather than through a copy/paste web interface (google.gemini.com), the previous EOF (Heredoc) mandate is deprecated. The AI MUST use its built-in, native file editing and creation tools (e.g., direct file writing/replacing mechanisms) to modify code. This is the easiest, fastest, and most reliable method available to the AI.

### 2. Versioning
- **Header:** Every source file MUST start with a version comment: `// Version: X.XX` (JS) or `# Version: X.XX` (Python).
- **Increment:** Increase the version by **0.01** for every edit.
