MCP Server API Reference

Overview

The everythingbot Master Control Program (MCP) server acts as the central hub, orchestrating communication between the Web UI, NLP Service, and various data storage mechanisms. It exposes a RESTful API for managing chat sessions, logging events, handling file operations, and initiating AI training processes.

Base URL: http://localhost:3000

System Logging

Logs system events from all components to the ai_only.db system_logs table.

POST /log

Records a general system log entry.

Request Body:
{
    "source": "string",  // e.g., "WEB_UI", "NLP_SERVICE", "MCP_SERVER"
    "level": "string",   // e.g., "INFO", "WARN", "ERROR", "FATAL"
    "message": "string",
    "details": {}        // JSON object for additional context
}

Responses:

200 OK: { "success": true }

User Quality Logging

Records user feedback (ratings) to the ai_only.db user_qualities table.

POST /log/user_quality

Records user-provided feedback on bot responses.

Request Body:
{
    "session_id": "integer",
    "quality": "string", // e.g., "good", "bad"
    "comment": "string"  // Optional: User's comment on the quality
}

Responses:

200 OK: { "success": true, "id": 1 }

Lessons Learned Logging

Records lessons learned by the AI to the ai_only.db lessons_learned table.

POST /log/lesson_learned

Records an AI-generated lesson.

Request Body:
{
    "session_id": "integer",
    "lesson": "string"   // The lesson learned by the AI
}

Responses:

200 OK: { "success": true, "id": 1 }

File Operations

Enables CRUD operations on files within the uploads/ directory.

POST /upload

Uploads a single file to the uploads/ directory.

Request Body:
multipart/form-data (field name: 'file')

Responses:

200 OK: { "success": true, "message": "File uploaded successfully.", "file": { ... } }

GET /files

Lists all files in the uploads/ directory.

Responses:

200 OK: { "files": ["file1.txt", "image.png"] }

GET /files/:filename

Retrieves the content of a specific file.

Responses:

200 OK: "File content as plain text"

PUT /files/:filename

Updates the content of an existing file or creates a new one.

Request Body:
{ "content": "string" }

Responses:

200 OK: { "success": true, "message": "File updated successfully." }

DELETE /files/:filename

Deletes a specific file from the uploads/ directory.

Responses:

200 OK: { "success": true, "message": "File deleted successfully." }

Browse Directory

Allows listing contents of directories within the project root.

GET /browse/:path?

Lists files and directories at the specified path (relative to project root). If path is omitted, lists project root.

Responses:

200 OK: { "path": "string", "contents": [{ "name": "string", "is_directory": boolean, "is_file": boolean, "path": "string" }] }

Chat and NLP Integration

POST /chat

Sends a message to the NLP service and logs the interaction. Used for main chat functionality.

Request Body:
{
    "message": "string",
    "session_id": "integer" // Current chat session ID
}

Responses:

200 OK: { "response": "string", "message_id": 1, "session_id": 1 }

Session Management

GET /sessions

Retrieves a list of all chat sessions.

Responses:

200 OK: { "sessions": [{ "id": 1, "created_at": "datetime", "name": "string" }] }

GET /history/:session_id

Retrieves chat history for a specific session.

Responses:

200 OK: { "history": [{ "id": 1, "session_id": 1, "timestamp": "datetime", "user_input": "string", "bot_response": "string", "event": "string", "rating": "string" }] }

POST /clear-history/:session_id

Clears chat history for a specific session.

Responses:

200 OK: { "success": true, "message": "Chat history cleared for session 1" }

Training

POST /rate

Records a rating for a specific bot message.

Request Body:
{
    "message_id": "integer",
    "rating": "string" // e.g., "good", "bad"
}

Responses:

200 OK: { "success": true, "message": "Message 1 rated good" }

POST /train

Initiates the AI training process using rated conversations.

Responses:

200 OK: { "success": true, "message": "Training initiated: ..." }
